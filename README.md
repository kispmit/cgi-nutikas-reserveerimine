# cgi-nutikas-reserveerimine

Testing: /src/main/java/com.tim.restaurant alt RestaurantApplication käivitada ja [localhost](http://localhost:8080/) toimumist jälgida.

Töö jaoks kulus mul ~7 tundi, millest suurem enamus läks backendile. Saadud tulemus ei ole kindlasti täielik ning vajaks veel nokitsemist, et parem välja näha. Lisaks oleks mõned funktsioonid vaja veel lisada, nt Broneeringu süsteem ise töötab kahtlaselt - broneerimise järel ei uuene laua olek frontendis alati korrektselt, võib jääda mulje, et laud on endiselt saadaval, kuigi ta on juba broneerituks backendis muudetud. Sama lauda uuesti broneerides saab popup errori. Selle asemel oleks palju parem, kui UI ise ennast rohelisest lauast punaseks lauaks paneks.
Probleeme tekkis piisavalt, millele lahendused leidsin AI-lt abi paludes ning kahel korral ka w3schoolsi lehelt.
AI tööriistasid on mõõdukalt igal pool kasutatud, ainus pikem koodijupp, mis on täielikult copy pasted on index.html file, mille tegid ChatGPT ja Claude koostöös :). Ülejäänud koodis olen AI-d kasutanud kontrollimiseks, ideede genereerimiseks ja pisivigade parandamiseks.
